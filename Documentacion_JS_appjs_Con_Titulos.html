🏗️ ESTADO UNIFICADO Y REFERENCIAS AL DOM

const state Objeto de Estado Unificado de la Aplicación
¿Qué hace?
Es el Single State Tree (Árbol de Estado Único) de la aplicación. Contiene toda la información que la app necesita para funcionar: citas, filtros y flags de UI.
¿Dónde?
Declarado en la sección I, al inicio del archivo, después del bloque de comentarios de documentación.
Propiedades
  • citas: Array[] — Lista de objetos cita {id, paciente, servicio, fecha, hora}
  • filtroBusqueda: String — Texto del buscador para filtrar pacientes
  • idCitaParaEliminar: Number|null — ID de la cita en proceso de eliminación
  • tipoDeLimpieza: String — 'citas' (limpiar todo) o 'cita' (eliminar una)
¿Por qué?
Centralizar el estado evita inconsistencias. Cualquier cambio en state.citas dispara un renderizado completo, garantizando que la UI siempre refleje los datos reales.
Patrón
Single State Tree + Observer-like Update Pattern. La UI reacciona y se repinta ante cambios en el estado.
const state = { citas: [], filtroBusqueda: '', idCitaParaEliminar: null, tipoDeLimpieza: 'cita' };
DOM Refs Referencias a Elementos del DOM (Cacheo de Selectores)
¿Qué hace?
Cachea (guarda en memoria) todas las referencias a elementos del DOM que el JavaScript necesita manipular. Evita llamar repetidamente a document.getElementById() en cada función.
¿Dónde?
Sección I, justo después del objeto state. Se organizan por áreas funcionales.
Grupos
  • Formulario Creación: formularioCita, inputPaciente, inputServicio, inputFecha, inputHora, alertaFechaFinSemana
  • Dashboard: statTotalValor, statHoyValor, statProximaValor, statPopularValor
  • Búsqueda y Lista: inputBuscador, botonExportar, botonLimpiarTodo, contenedorListaCitas
  • Modal Edición: modalEdicionOverlay, formularioEdicion, inputEditId, inputEditPaciente, inputEditServicio, inputEditFecha, inputEditHora, alertaEditFechaFinSemana, botonCerrarModal, botonCancelarModal
  • Confirmación: modalConfirmacionOverlay, tituloConfirmacion, descripcionConfirmacion, botonAceptarConfirmacion, botonCancelarConfirmacion
  • Toasts y Tema: contenedorToasts, botonAlternarTema
¿Por qué?
Optimización de rendimiento. getElementById es rápido pero repetirlo en cada render es innecesario. Cachear las referencias es una práctica estándar en apps state-driven.
// Formulario de Creación (CREATE) const formularioCita = document.getElementById('cita-form'); const inputPaciente = document.getElementById('paciente'); // ... y 20+ referencias más organizadas por áreas

⚡ INICIALIZACIÓN DE LA APLICACIÓN

DOMContentLoaded Punto de Entrada Principal (App Bootstrapping)
¿Qué hace?
Es el Main Entry Point de la aplicación. Se ejecuta cuando el navegador ha cargado y analizado completamente el HTML inicial.
¿Dónde?
Sección II, registrado como listener del evento document.addEventListener('DOMContentLoaded', ...).
Flujo de inicialización
  1. Inicializa tema visual (claro/oscuro)
  2. Carga citas desde localStorage al estado
  3. Establece fecha mínima en los inputs de fecha
  4. Renderiza la interfaz por primera vez
  5. Muestra toast de bienvenida "Sistema listo y operando 100% fuera de línea"
¿Por qué?
Garantiza que el DOM esté listo antes de manipularlo. Evita errores de elementos no encontrados.
API del Navegador
document.addEventListener() — Registra un callback que se ejecuta cuando el evento DOMContentLoaded se dispara.
document.addEventListener('DOMContentLoaded', () => { inicializarTemaClaroOscuro(); cargarCitasDesdeAlmacenamiento(); establecerLimiteFechaMinima(); renderizarInterfaz(); mostrarToast('Sistema listo y operando 100% fuera de línea.', 'success'); });
establecerLimiteFechaMinima() Establecer Fecha Mínima en los Inputs de Fecha
¿Qué hace?
Calcula la fecha actual en formato local AAAA-MM-DD y la asigna como propiedad min de los inputs de fecha (creación y edición). Esto deshabilita fechas pasadas en el selector nativo.
¿Dónde?
Sección II. Llamada desde el punto de entrada.
Lógica clave
Usa new Date() + getFullYear(), getMonth() (0-11), getDate() + String(...).padStart(2, '0') para formatear correctamente sin problemas de huso horario UTC.
¿Por qué?
Evita que el usuario agende citas en fechas pasadas. Usa el huso horario local para no restar un día por desfase UTC.
Relaciones
Modifica inputFecha.min y inputEditFecha.min.
function establecerLimiteFechaMinima() { const hoy = new Date(); const anio = hoy.getFullYear(); const mes = String(hoy.getMonth() + 1).padStart(2, '0'); const dia = String(hoy.getDate()).padStart(2, '0'); const fechaMinima = `${anio}-${mes}-${dia}`; inputFecha.min = fechaMinima; inputEditFecha.min = fechaMinima; }

🌗 CONTROL DEL TEMA VISUAL

inicializarTemaClaroOscuro() Inicializar y Alternar Tema Claro/Oscuro
¿Qué hace?
Lee la preferencia de tema desde localStorage. Si no existe, detecta la preferencia del sistema operativo mediante window.matchMedia() y aplica el tema al elemento raíz <html>.
¿Dónde?
Sección III. Llamada en el punto de entrada y en el listener del botón de tema.
APIs utilizadas
  • localStorage.getItem() — Lee preferencia guardada
  • window.matchMedia('(prefers-color-scheme: dark)') — Detecta preferencia del SO
  • document.documentElement.setAttribute() — Aplica tema al <html>
¿Por qué?
Persiste la preferencia del usuario entre sesiones. Respeta la configuración del sistema operativo como fallback.
Relaciones
El botón botonAlternarTema tiene un listener que invierte el tema actual y lo persiste.
function inicializarTemaClaroOscuro() { const temaGuardado = localStorage.getItem('odontologia-tema'); if (temaGuardado) { document.documentElement.setAttribute('data-theme', temaGuardado); } else { const prefiereOscuro = window.matchMedia('(prefers-color-scheme: dark)').matches; const tema = prefiereOscuro ? 'dark' : 'light'; document.documentElement.setAttribute('data-theme', tema); localStorage.setItem('odontologia-tema', tema); } }

💾 PERSISTENCIA DE DATOS (LocalStorage)

cargarCitasDesdeAlmacenamiento() Cargar Citas desde localStorage al Estado
¿Qué hace?
Lee la clave 'odontologia-citas' del localStorage, parsea el JSON y lo asigna a state.citas. Si no hay datos o el JSON está corrupto, inicializa un array vacío.
¿Dónde?
Sección IV. Llamada en el punto de entrada.
Manejo de errores
Usa try/catch para capturar errores de parsing JSON. Si los datos están corruptos, muestra un toast de error y limpia el almacenamiento.
APIs utilizadas
localStorage.getItem() — Lectura; JSON.parse() — Deserialización; JSON.stringify() — Serialización (en guardar).
¿Por qué?
Permite que las citas persistan tras cerrar el navegador. Funciona 100% offline sin servidor.
function cargarCitasDesdeAlmacenamiento() { try { const rawData = localStorage.getItem('odontologia-citas'); state.citas = rawData ? JSON.parse(rawData) : []; } catch (error) { console.error("Error al analizar datos:", error); mostrarToast("Error al cargar citas corruptas. Base limpia.", "error"); state.citas = []; guardarCitasEnAlmacenamiento(); } }
guardarCitasEnAlmacenamiento() Guardar Citas del Estado en localStorage
¿Qué hace?
Serializa state.citas a JSON y lo guarda en localStorage bajo la clave 'odontologia-citas'.
¿Dónde?
Sección IV. Llamada después de cada operación CRUD (crear, editar, eliminar).
Manejo de errores
Captura errores de escritura (ej. almacenamiento lleno) y muestra toast informativo.
¿Por qué?
Garantiza que cualquier cambio en el estado se persista inmediatamente. Es la capa de persistencia de la app.
function guardarCitasEnAlmacenamiento() { try { const dataString = JSON.stringify(state.citas); localStorage.setItem('odontologia-citas', dataString); } catch (error) { console.error("Error al escribir datos:", error); mostrarToast("No se pudo guardar. Espacio lleno.", "error"); } }

🎨 RENDERIZADO Y DASHBOARD

renderizarInterfaz() Pipeline Central de Renderizado (Central Render Pipeline)
¿Qué hace?
Es la función principal de renderizado. Limpia y reconstruye el listado de citas y actualiza los indicadores del dashboard. Actúa como el "motor" de la UI.
¿Dónde?
Sección V. Llamada después de cada modificación del estado (CRUD, búsqueda).
Pasos del pipeline
  1. filtrarCitasActualmente() — Filtra según búsqueda
  2. ordenarCitasCronologicamente() — Ordena fecha/hora ascendente
  3. renderizarTarjetasCita() — Inyecta tarjetas en el DOM
  4. actualizarEstadisticasDashboard() — Actualiza métricas
¿Por qué?
Un único punto de renderizado garantiza consistencia. No importa qué operación modifique el estado, la UI siempre se repinta correctamente.
function renderizarInterfaz() { const citasFiltradas = filtrarCitasActualmente(); ordenarCitasCronologicamente(citasFiltradas); renderizarTarjetasCita(citasFiltradas); actualizarEstadisticasDashboard(); }
filtrarCitasActualmente() Filtrar Citas según el Buscador (Live Filter)
¿Qué hace?
Filtra el array state.citas comparando el nombre del paciente con el texto de state.filtroBusqueda. La comparación es insensible a mayúsculas/minúsculas.
¿Dónde?
Sección V. Llamada dentro de renderizarInterfaz().
Retorno
Array de citas filtradas (copia con spread [...state.citas] si no hay búsqueda).
¿Por qué?
La búsqueda en tiempo real permite encontrar citas rápidamente sin recargar la página.
function filtrarCitasActualmente() { const query = state.filtroBusqueda.trim().toLowerCase(); if (query === '') return [...state.citas]; return state.citas.filter(cita => cita.paciente.toLowerCase().includes(query) ); }
ordenarCitasCronologicamente() Ordenar Citas Cronológicamente
¿Qué hace?
Ordena un array de citas por fecha (más antigua a más nueva) y, si la fecha coincide, por hora. Modifica el array por referencia.
¿Dónde?
Sección V. Llamada dentro de renderizarInterfaz() y antes de exportar CSV.
Método de ordenamiento
Usa array.sort() con comparación lexicográfica de strings (localeCompare()). Las fechas en formato AAAA-MM-DD y horas en HH:MM permiten comparación directa de strings.
¿Por qué?
Las citas se muestran en orden cronológico para facilitar la planificación del consultorio.
function ordenarCitasCronologicamente(arrayCitas) { arrayCitas.sort((a, b) => { if (a.fecha !== b.fecha) { return a.fecha.localeCompare(b.fecha); } return a.hora.localeCompare(b.hora); }); }
renderizarTarjetasCita() Inyectar Tarjetas de Citas en el DOM
¿Qué hace?
Limpia el contenedor de citas y reconstruye las tarjetas HTML dinámicamente. Cada tarjeta incluye nombre del paciente, servicio (con badge de color), fecha formateada, hora formateada, y botones de editar/eliminar.
¿Dónde?
Sección V. Llamada desde renderizarInterfaz().
Técnicas utilizadas
  • document.createElement() — Crea nodos DOM
  • innerHTML — Inyecta markup con template literals
  • contenedor.appendChild() — Agrega tarjetas al DOM
  • escaparHTML() — Sanitiza nombres de pacientes para evitar XSS
  • escaparJS() — Escapa comillas para atributos onclick
Empty State
Si no hay citas, muestra un estado vacío con icono SVG, título "No se encontraron citas" y mensaje contextual según si hay citas en el estado o no.
¿Por qué?
La generación dinámica permite que la UI se adapte a cualquier cantidad de citas. La sanitización es crítica para seguridad.
function renderizarTarjetasCita(citasAMostrar) { contenedorListaCitas.innerHTML = ''; if (citasAMostrar.length === 0) { // Renderizar empty state return; } citasAMostrar.forEach(cita => { const tarjeta = document.createElement('div'); tarjeta.className = `appointment-card service-${cita.servicio.toLowerCase()}`; // ... innerHTML con datos sanitizados contenedorListaCitas.appendChild(tarjeta); }); }
actualizarEstadisticasDashboard() Calcular y Actualizar Estadísticas del Dashboard
¿Qué hace?
Calcula 4 métricas agregadas sobre state.citas y actualiza los 4 indicadores del dashboard en tiempo real.
¿Dónde?
Sección V. Llamada desde renderizarInterfaz().
Métricas calculadas
  • Total Citas: state.citas.length
  • Citas Hoy: Filtra citas donde c.fecha === hoyStr
  • Próxima Cita: Filtra citas futuras (fecha > hoy, o fecha=hoy y hora >= actual), ordena y toma la primera
  • Servicio Más Solicitado: Calcula la moda estadística contando ocurrencias de cada servicio
¿Por qué?
Proporciona al usuario una visión rápida del estado del consultorio sin necesidad de contar manualmente.
function actualizarEstadisticasDashboard() { const hoyStr = obtenerFechaHoyLocal(); statTotalValor.textContent = state.citas.length; // ... cálculos de citas hoy, próxima cita, servicio popular }

➕ CREAR CITA (CREATE)

submit #cita-form Listener de Creación de Nueva Cita
¿Qué hace?
Escucha el evento submit del formulario principal, valida los datos, verifica reglas de negocio y crea una nueva cita en el estado.
¿Dónde?
Sección VI. Listener registrado sobre formularioCita.
Validaciones aplicadas
  1. Campos obligatorios: Verifica que paciente, servicio, fecha y hora no estén vacíos
  2. Fines de semana: Usa new Date(fecha).getUTCDay() (0=Domingo, 6=Sábado). Muestra advertencia visual
  3. Choque de horarios (Double Booking): Busca si ya existe una cita a la misma fecha y hora
Generación de ID
Usa Date.now() como clave primaria única (timestamp de milisegundos).
Flujo tras validación
Crea objeto cita → state.citas.push() → guardarCitasEnAlmacenamiento() → formularioCita.reset() → renderizarInterfaz() → mostrarToast()
formularioCita.addEventListener('submit', (e) => { e.preventDefault(); // 1. Extraer valores // 2. Validar campos obligatorios // 3. Validar fin de semana (getUTCDay) // 4. Validar choque de horarios (find) // 5. Crear cita con id: Date.now() // 6. Persistir y renderizar });

¿Qué hace?

inputFecha.addEventListener('input', (e) => { const fechaSeleccionada = e.target.value; const diaSemana = new Date(fechaSeleccionada).getUTCDay(); alertaFechaFinSemana.style.display = (diaSemana === 0 || diaSemana === 6) ? 'flex' : 'none'; });

✏️ ACTUALIZAR CITA (UPDATE)

abrirModalEdicion(id) Abrir Modal de Edición con Datos de la Cita
¿Qué hace?
Busca la cita por ID en el estado, carga sus datos en los inputs del modal de edición y muestra el modal agregando la clase .show.
¿Dónde?
Sección VII. Llamada desde el onclick de cada botón de editar en las tarjetas.
Parámetro
id — Clave primaria (timestamp) de la cita a editar.
Acciones
  • Busca cita con state.citas.find(c => c.id === id)
  • Carga valores en inputs del modal (paciente, servicio, fecha, hora)
  • Verifica si la fecha es fin de semana para mostrar advertencia

¿Por qué?
El focus lock (foco atrapado) es una práctica de accesibilidad (A11y) que mantiene el teclado dentro del modal.
function abrirModalEdicion(id) { const cita = state.citas.find(c => c.id === id); if (!cita) return; inputEditId.value = cita.id; inputEditPaciente.value = cita.paciente; // ... cargar demás campos modalEdicionOverlay.classList.add('show'); inputEditPaciente.focus(); }

📤 EXPORTACIÓN DE REPORTES

click #exportar-btn Generar y Descargar Reporte CSV
¿Qué hace?
Genera un archivo CSV con todas las citas ordenadas cronológicamente, incluye BOM UTF-8 para compatibilidad con Excel en español, y fuerza la descarga local sin servidor.
¿Dónde?
Sección X. Listener del botón "Exportar Reporte".
APIs del Navegador
  • new Blob() — Crea objeto binario/texto en memoria RAM
  • URL.createObjectURL() — Genera URL temporal vinculada al Blob
  • URL.revokeObjectURL() — Libera memoria del Blob tras la descarga
Seguridad CSV
Escapa comillas dobles en nombres de pacientes (replace(/"/g, '""')) para evitar inyección de fórmulas CSV. Usa comillas alrededor de cada campo.
BOM UTF-8
Prefija "\ufeff" al contenido para que Excel reconozca correctamente caracteres especiales (tildes, eñes).
const archivoBlob = new Blob([contenidoCsv], { type: 'text/csv;charset=utf-8;' }); const enlace = document.createElement('a'); const url = URL.createObjectURL(archivoBlob); enlace.href = url; enlace.download = `reporte_citas_${obtenerFechaHoyLocal()}.csv`; enlace.click(); URL.revokeObjectURL(url);

🛠️ FUNCIONES AUXILIARES (HELPERS)

obtenerFechaHoyLocal() Obtener Fecha Actual en Formato Local
¿Qué hace?
Devuelve la fecha actual en formato AAAA-MM-DD usando el huso horario local del cliente.
Retorno
String en formato 'AAAA-MM-DD'.
¿Por qué?
Evita el desfase UTC que ocurre con toISOString(), que puede restar un día en zonas horarias americanas.
function obtenerFechaHoyLocal() { const hoy = new Date(); return `${hoy.getFullYear()}-${String(hoy.getMonth()+1).padStart(2,'0')}-${String(hoy.getDate()).padStart(2,'0')}`; }
formatearFechaEspanol() Formatear Fecha a Español Legible
¿Qué hace?
Convierte una fecha en formato AAAA-MM-DD a texto legible en español (ej: "Lunes, 22 de junio de 2026").
Técnica
Divide el string manualmente y usa new Date(anio, mes, dia) para evitar desfases UTC. Luego aplica toLocaleDateString('es-ES', ...) con opciones de día de semana, día numérico, mes y año.
Capitalización
Capitaliza la primera letra del resultado con charAt(0).toUpperCase() + slice(1).
function formatearFechaEspanol(fechaTexto) { const partes = fechaTexto.split('-'); const fechaObj = new Date(parseInt(partes[0]), parseInt(partes[1])-1, parseInt(partes[2])); const opciones = { weekday: 'long', day: 'numeric', month: 'long', year: 'numeric' }; let f = fechaObj.toLocaleDateString('es-ES', opciones); return f.charAt(0).toUpperCase() + f.slice(1); }
formatearHora12() Formatear Hora a Formato 12 Horas (AM/PM)
¿Qué hace?
Convierte una hora en formato HH:MM (24h) a formato HH:MM AM/PM (12h).
Lógica
Divide el string por ':', calcula AM/PM, convierte horas 0→12 y 13-23→1-11, y formatea con padStart(2, '0').
function formatearHora12(horaTexto) { const partes = horaTexto.split(':'); let horas = parseInt(partes[0]); const ampm = horas >= 12 ? 'PM' : 'AM'; horas = horas % 12; horas = horas ? horas : 12; return `${String(horas).padStart(2,'0')}:${partes[1]} ${ampm}`; }
escaparHTML() Sanitización contra XSS (Cross-Site Scripting)
¿Qué hace?
Reemplaza caracteres especiales HTML por sus entidades correspondientes para evitar inyección de código malicioso.
Caracteres escapados
& → &, < → <, > → >, " → ", ' → ', / → /, ` → `
¿Por qué?
Los nombres de pacientes se inyectan en innerHTML. Sin sanitización, un nombre como "<script>alert('hacked')</script>" ejecutaría código.
function escaparHTML(texto) { const mapa = { '&': '&', '<': '<', '>': '>', '"': '"', "'": ''', '/': '/', '`': '`' }; return texto.replace(/[&<>"'`\/]/g, m => mapa[m]); }
escaparJS() Escapar Comillas para Atributos onclick
¿Qué hace?
Escapa comillas simples en nombres de pacientes para evitar romper los atributos onclick de los botones de eliminar.
¿Por qué?
Un nombre como "O'Brien" rompería el string JavaScript: onclick="solicitarEliminarCita(1, 'O'Brien')". Escapando a "O\'Brien" se mantiene la sintaxis válida.
function escaparJS(texto) { return texto.replace(/'/g, "\\'"); }

mostrarToast(mensaje, tipo) Crear y Mostrar Notificaciones Toast
¿Qué hace?
Crea dinámicamente un elemento toast con icono SVG, mensaje sanitizado, y lo inyecta en el contenedor flotante. Se auto-destruye tras 4 segundos con animación CSS.
¿Dónde?
Sección XII. Función auxiliar llamada desde todo el sistema.
Parámetros
  • mensaje — Texto de la notificación (sanitizado con escaparHTML())
  • tipo — 'success' (verde), 'error' (rojo), 'warning' (naranja), 'info' (azul)
Iconos SVG
Cada tipo tiene un icono SVG distinto: check circular (success), cruz (error), triángulo de advertencia (warning), letra i (info).
Auto-destrucción
