🏗️ ESTRUCTURA RAÍZ Y METADATOS

<!DOCTYPE html> Declaración del Tipo de Documento
¿Qué hace?
Declara que el documento es HTML5. Permite al navegador renderizar la página en modo estándar sin compatibilidad hacia atrás.
¿Dónde?
Primera línea del archivo, antes de <html>.
¿Por qué?
Sin DOCTYPE, el navegador entraría en "quirks mode" y los estilos CSS se comportarían de forma inconsistente.
Relaciones
Precede a <html> que contiene todo el documento.
<!-- Primera línea obligatoria del documento --> <!DOCTYPE html>
<html lang="es"> Elemento Raíz del Documento
¿Qué hace?
Elemento raíz del documento. Define el idioma principal como español (es) para lectores de pantalla, motores de búsqueda y herramientas de traducción.
¿Dónde?
Envuelve todo el contenido después del DOCTYPE.
Atributos clave
lang="es" — Idioma español para accesibilidad y SEO.
¿Por qué?
Mejora la accesibilidad (lectores de pantalla pronuncian correctamente), el SEO (motores de búsqueda indexan el idioma) y la experiencia del usuario (traducciones automáticas).
Relaciones
Contiene <head> y <body>.
<html lang="es"> <!-- Contiene head y body --> </html>
<head> Cabecera del Documento
¿Qué hace?
Contiene metadatos, configuraciones y recursos externos que no se renderizan visualmente pero son esenciales para el funcionamiento de la página.
¿Dónde?
Dentro de <html>, antes de <body>.
¿Por qué?
Proporciona información al navegador sobre codificación, viewport, descripción, autor y recursos CSS/JS necesarios.
Relaciones
Contiene <meta>, <title>, <link> y el comentario de documentación.
<meta charset> Codificación de Caracteres UTF-8
¿Qué hace?
Especifica la codificación de caracteres UTF-8, que incluye todos los caracteres del español (tildes, eñes, signos de puntuación).
¿Dónde?
Primera etiqueta dentro de <head>.
¿Por qué?
Evita que caracteres especiales (á, é, í, ó, ú, ñ, ¿, ¡) se muestren como símbolos extraños.
Relaciones
Debe ser el primer <meta> para evitar problemas de decodificación.
<meta charset="UTF-8">
<meta viewport> Viewport Responsive
¿Qué hace?
Configura el viewport para que el ancho sea igual al ancho del dispositivo y la escala inicial sea 1.0 (sin zoom).
¿Dónde?
Segunda etiqueta <meta> dentro de <head>.
Atributos clave
width=device-width — Ancho igual al dispositivo; initial-scale=1.0 — Sin zoom inicial.
¿Por qué?
Hace que la página sea responsive y se vea correctamente en móviles, tablets y desktops sin zoom forzado.
Relaciones
Trabaja con las media queries de CSS (@media (max-width: 992px)).
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta description> Descripción SEO
¿Qué hace?
Proporciona una descripción de 150-160 caracteres que aparece en los resultados de búsqueda de Google (SERP).
¿Dónde?
Tercera etiqueta <meta> dentro de <head>.
Contenido
"Sistema de Gestión Odontológica Offline para el Consultorio Odontológico Alicia J. Colmenares S. en Barrio Obrero, San Cristóbal, Estado Táchira. Desarrollado como Proyecto Sociotecnológico I para el PNF en Informática."
¿Por qué?
Mejora el SEO (posicionamiento en buscadores) y ayuda a los usuarios a entender de qué trata la página antes de hacer clic.
Relaciones
Usada por motores de búsqueda (Google, Bing) para mostrar snippets.
<meta name="description" content="Sistema de Gestión Odontológica Offline...">
<meta author> Autor del Documento
¿Qué hace?
Identifica al autor del documento HTML.
¿Dónde?
Cuarta etiqueta <meta> dentro de <head>.
¿Por qué?
Proporciona atribución del desarrollador y puede usarse para créditos o contacto.
Relaciones
Complementa la información del footer sobre el autor.
<meta name="author" content="Ivor Alexander Guzmán Z.">
<title> Título de la Página
¿Qué hace?
Define el título que aparece en la pestaña del navegador, en los marcadores/favoritos y en los resultados de búsqueda.
¿Dónde?
Dentro de <head>, después de los <meta>.
Contenido
"Panel de Gestión Odontológica - Dra. Alicia J. Colmenares S."
¿Por qué?
Es el identificador principal de la página para el usuario y los motores de búsqueda. Debe ser descriptivo y conciso.
Relaciones
Aparece en la pestaña del navegador y en resultados de búsqueda.
<title>Panel de Gestión Odontológica - Dra. Alicia J. Colmenares S.</title>
<!-- --> Comentario de Documentación
¿Qué hace?
Bloque de comentario HTML que sirve como guía rápida para desarrolladores que lean el código fuente. Explica el propósito, funcionamiento y dependencias del archivo.
¿Dónde?
Dentro de <head>, antes del <link> de CSS.
Contenido
Explica qué es el archivo, qué hace, cómo lo hace y notas sobre dependencias con app.js y style.css.
¿Por qué?
Facilita el mantenimiento y la comprensión del código para otros desarrolladores o para el mismo autor en el futuro.
Relaciones
Referencia a style.css y app.js.
<!-- Sistema de Gestión Odontológica - Documentación Propósito: Gestión offline de citas médicas Dependencias: style.css, app.js -->
<link> Hoja de Estilos CSS
¿Qué hace?
Vincula el archivo CSS externo que contiene todos los estilos, variables y animaciones de la aplicación.
¿Dónde?
Último elemento dentro de <head>, antes de cerrar </head>.
Atributos clave
rel="stylesheet" — Indica que es una hoja de estilos; href="style.css" — Ruta relativa al archivo CSS.
¿Por qué?
Separa la presentación (CSS) del contenido (HTML), siguiendo las mejores prácticas de desarrollo web.
Relaciones
El archivo style.css define las 80 clases CSS documentadas en los PDFs.
<link rel="stylesheet" href="style.css">

📄 CUERPO Y CONTENEDOR PRINCIPAL

<body> Cuerpo del Documento
¿Qué hace?
Contiene todo el contenido visible de la página: texto, imágenes, formularios, botones, modales, etc.
¿Dónde?
Dentro de <html>, después de </head>.
¿Por qué?
Es el contenedor de todos los elementos que el usuario puede ver e interactuar.
Relaciones
Contiene .app-container, que a su vez contiene todo el layout de la aplicación.
<body> <!-- Todo el contenido visible va aquí --> </body>
<div class="app-container"> Contenedor Principal de la Aplicación
¿Qué hace?
Contenedor raíz que envuelve TODA la aplicación. Limita el ancho máximo a 1280px, centra horizontalmente y añade padding. Usa flexbox en columna.
¿Dónde?
Primer hijo de <body>, envuelve todo el contenido excepto <script>.
Clase CSS
.app-container (documentada en los PDFs de clases CSS).
¿Por qué?
Sin este contenedor, la página se vería pegada a los bordes y muy ancha en monitores grandes. Proporciona el marco visual.
Relaciones
Contiene: .main-header, .stats-grid, .main-grid, modales, .toast-container, .main-footer.
<div class="app-container"> <!-- Header, Stats, Grid, Modales, Footer --> </div>
